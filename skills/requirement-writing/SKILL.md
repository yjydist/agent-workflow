# Skill: Requirement Writing

## Goal

把模糊需求写成可实现,可测试的功能需求.

## Requirement Unit Template

```md
### 功能名称

#### Actor

#### Preconditions

#### Input

#### Rules

#### Success Result

#### Failure Cases

#### Acceptance
```

## Rules

- 避免"支持""优化""完善"这类空泛词.
- 每个功能都要写失败情况.
- 每个需求都应该能追踪到一个或多个验收项.
- 不确定内容必须标记 TODO.
