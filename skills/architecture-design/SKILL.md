# Skill: Architecture Design

## Goal

为项目选择足够简单,可实现,可维护的架构.

## Output Template

```md
# Architecture

## System Context

## Modules

## Dependency Direction

## Runtime Flow

## Technology Choices

## Explicit Non-goals

## Risks
```

## Rules

- 优先单体,模块化,清晰分层.
- 不为了简历亮点强行引入复杂架构.
- 架构必须能映射到 implementation-plan.
