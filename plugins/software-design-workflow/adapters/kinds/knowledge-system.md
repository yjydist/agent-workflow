# Knowledge System adapter

## Layer

Kind

## Applies when

Use when the software manages, retrieves, transforms, or reasons over documents, concepts, references, or organizational knowledge.

## Design focus

Define knowledge sources, metadata, ingestion, retrieval, freshness, permissions, provenance, and answer or output quality.

## Expected artifacts

Knowledge model, source inventory, retrieval workflows, quality evaluation, governance notes.

## Legacy migration

Legacy `rag-app` can be modeled as `knowledge-system` plus `http-api`, `web-ui`, or `agent-tools`, with AI and retrieval concerns added explicitly.
