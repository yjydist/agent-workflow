# Library SDK adapter

## Layer

Kind

## Applies when

Use when the software is consumed by developers through code imports, packages, or SDK calls.

## Design focus

Define public API stability, types, examples, compatibility policy, versioning, packaging, and support boundaries.

## Expected artifacts

Public API contract, usage examples, type model, compatibility matrix, versioning policy.

## Legacy migration

Legacy `library-sdk` remains a kind adapter and should pair with `library-api` plus the relevant runtime adapters.
