# Releases

This directory contains target-scoped docs. A target can be a version, milestone, release, experiment, or implementation slice.

## Placeholder target

`current-target/` is a template placeholder. Before active use, replace it with a concrete target id such as:

- `v1`
- `mvp`
- `release-2026-04`
- `milestone-1`

After replacement, update links in handoff docs and active planning notes to point to the concrete target directory.

## Target directory contents

Each target should include scope baseline, requirements, risks, implementation plan, review notes, and change log.
